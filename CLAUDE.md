## Working Rules
Before making UI, SEO, layout, or copy changes:
1. Read `docs/css-style.md`
2. Follow the approved colors, typography, buttons, layout, SEO, and copy rules
3. Do not introduce new colors, gradients, fonts, or generic agency copy
4. Keep changes focused and minimal unless asked for a full redesign

## Development Commands
- `npm run dev`
- `npm run build`
- `npm run preview`

## Before Finishing
- Check mobile responsiveness
- Run `npm run build`
- Explain which files changed
- Mention anything that still needs manual review

## Instruction Loading Rules

- Always read this file first before making any changes.
- If the task involves UI, design, layout, SEO, or copy:
  → Read `docs/css-style.md`
- Follow `docs/css-style.md` as the single source of truth for:
  colors, typography, buttons, layout, SEO, and copywriting
- Do not override or duplicate rules from `docs/css-style.md`
